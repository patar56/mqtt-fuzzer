   30 ### V1 — Unauthorized Will Message Exploitation
   31 **Severity:** HIGH | **Confidence:** HIGH | **CVSS Estimate:** 7.5
   32 **Source:** Burglars' IoT Paradise §V.A (IEEE S&P 2020)
   33
   34 **Description:** MQTT's Will message feature allows a client to register
 a
      message that the broker publishes on its behalf when it disconnects
 ungrace
      fully. On a broker with no ACL, an attacker can set a Will targeting any
 to
      pic they cannot directly publish to and then force-disconnect to trigger
 de
      livery, effectively bypassing topic-level publish controls.
   35
   36 **Evidence Collected:**
   37 - Attacker connected specifying `will_topic='restricted/admin'`,
 `will_mess
      age=b'UNAUTHORIZED_WILL_FROM_ATTACKER'`
   38 - TCP connection closed without sending DISCONNECT packet
   39 - Observer subscriber on `restricted/admin` received the Will payload:
 `b'U
      NAUTHORIZED_WILL_FROM_ATTACKER'`
   40 - Same attack confirmed on `commands/all` and `alerts/critical`
   41 - Note: Mosquitto correctly blocked Will delivery to `$SYS/test` (that
 name
      space is protected)
   42
   43 **Broker Log Evidence:**
   44 ```
   45 Received PUBLISH from mqtt_agent_fuzz (d0, q0, r1, m0, 'commands/all',
 ...
      (17 bytes))
   46 ```
   47
   48 **Reproduction Steps:**
   49 1. `CONNECT client_id='will_attacker', will_topic='restricted/admin',
 will_
      message='ATTACK', clean_session=True`
   50 2. Receive `CONNACK rc=0x00` (broker accepts)
   51 3. Close TCP socket without sending `DISCONNECT` (raw `socket.close()`)
   52 4. Observer subscribed to `restricted/admin` receives the payload
   53 5. Attacker has published to a topic without using `PUBLISH` — bypasses
 any
       topic-level publish ACL
   54
   55 **Root Cause:** No ACL file configured (`acl_file` directive commented
 out)
      . With ACL enabled and `will_topic` validation, brokers can restrict
 which
      Will topics a given ClientID may use.
   56
   57 **CWE:** CWE-284 (Improper Access Control)
   58
   59 **Mitigation:** Enable ACL with explicit per-client topic permissions.
 Add
      Will topic validation at connect time. Restrict anonymous access.

  103 - Victim connected with `ClientID='victim_device_001'`,

  296 ### Generation-Based Campaign (85 test cases)
  297 - **Test case categories:** CONNECT variants (29), PUBLISH variants (23),
  S
      UBSCRIBE variants (18), Malformed packets (15)
  298 - **Anomaly rate:** 22/85 = 25.9%
  299 - **Primary anomaly type:** NO_RESPONSE (all 22 cases) — broker silently
 dr
      ops malformed connections rather than sending error responses
  300 - **Interpretation:** Mosquitto's "fail-silent" behavior on malformed
 packe
      ts is intentional and correct per spec. The broker closes TCP without a
 pro
      tocol-level error response to avoid information leakage about its parsing
  l
      ogic.
  301
  302 **High-value findings within generation fuzzing:**
  303 - Invalid protocol names (`mqtt` lowercase, `MQTT5`, empty string) →
 Silent
       TCP close, NO CONNACK
  304 - Will message with `$SYS/#` as topic → Connect rejected (broker refused
 th
      e CONNECT packet entirely)
  305 - Invalid topic filters in SUBSCRIBE (`a/#/b`, `#a`, `a#`) → Broker
 disconn
      ects with "malformed packet"
  306 - Wildcard QoS values in SUBSCRIBE (QoS=3, 127, 255) → Broker disconnects
  307 - All truncated packet variants (1/2/4 bytes) → Broker drops connection
 wit
      hout response

  309 ### Mutation-Based Campaign (200 test cases, 4 seeds × 50 mutations)
  310 - **Seeds used:** valid_connect, valid_subscribe, valid_publish_qos0,

 boundary_by

  312 - **Anomaly rate:** 96/200 = 48.0%
  313 - **All anomalies:** NO_RESPONSE type — mutations corrupted CONNECT
 header

  314 - **No crashes detected** — Mosquitto's C implementation handles all

  315 - **Notable:** Even the most aggressive mutations (4-byte flips, 4-byte

  375 ## Methodology
  376
  377 ### Phase 1: Grammar-Based Generation Fuzzing
  378 Implemented as `GenerationFuzzer` in `agent/fuzzing/engine.py`, inspired
 by
       the FUME fuzzer (Situ et al., 2022). Test cases were constructed from
 the
      formal MQTT packet grammar with systematic boundary value analysis
 across:
  379 - ClientID length boundaries (0, 1, 23, 24, 65535 bytes)
  380 - Protocol version/name variants
  381 - QoS boundary values (0, 1, 2, 3, 127, 255)
  382 - Topic string edge cases (empty, wildcards, $SYS namespace, null bytes)
  383 - Will message variants including restricted topic targets
  384 - Truncated and zero-remaining-length packet variants
  385

    70     props += bytes([0x11]) + struct.pack("!I", session_expiry)
    71
    72     # Receive Maximum (0x21)
    73     props += bytes([0x21]) + struct.pack("!H", receive_max)
    74
    75     # Maximum Packet Size (0x27)
    76     if max_packet_size is not None:
    77         props += bytes([0x27]) + struct.pack("!I", max_packet_size)
    78
    79     # Topic Alias Maximum (0x22)
    80     props += bytes([0x22]) + struct.pack("!H", topic_alias_max)
    81
    82     # User Properties (0x26)
    83     if user_props:
    84         for k, v in user_props:
    85             props += bytes([0x26]) + encode_utf8_string(k) +
 encode_utf8_s
       tring(v)
    86
    87     # Authentication Method (0x15)
    88     if auth_method is not None:
    89         props += bytes([0x15]) + encode_utf8_string(auth_method)
    90
    91     # Authentication Data (0x16)
    92     if auth_data is not None:
    93         props += bytes([0x16]) + struct.pack("!H", len(auth_data)) +
 auth_
       data
    94
    95     props_len = encode_remaining_length(len(props))
    96
    97     # Connect flags
    98     connect_flags = 0
    99     if clean_start:
   100         connect_flags |= 0x02
   101     if will_topic is not None:
   102         connect_flags |= 0x04
   103         connect_flags |= (will_qos & 0x03) << 3
   104         if will_retain:
   105             connect_flags |= 0x20
   106     if password is not None:
   107         connect_flags |= 0x40
   108     if username is not None:
   109         connect_flags |= 0x80
   110
   111     variable_header = (
   112         encode_utf8_string("MQTT")      # Protocol name
   113         + bytes([0x05])                  # Protocol level = 5
   114         + bytes([connect_flags])
   115         + struct.pack("!H", keepalive)
   116         + props_len + bytes(props)
   117     )
   118
   119     # Payload
   120     payload = encode_utf8_string(client_id)
   121     if will_topic is not None:
   122         # Will properties (empty for now)
   123         payload += bytes([0x00])  # will properties length = 0
   124         payload += encode_utf8_string(will_topic)
   125         wp = will_payload or b""
   126         payload += struct.pack("!H", len(wp)) + wp
   127     if username is not None:
   128         payload += encode_utf8_string(username)
   129     if password is not None:
   130         payload += struct.pack("!H", len(password)) + password
   131
   132     remaining = variable_header + payload
   133     fixed_header = bytes([0x10]) +
 encode_remaining_length(len(remaining))
   134     return fixed_header + remaining

   137 def build_publish_v5(
   138     topic: str,
   139     payload: bytes = b"",
   140     qos: int = 0,
   141     retain: bool = False,
   142     topic_alias: Optional[int] = None,
   143     user_props: Optional[List[Tuple[str, str]]] = None,
   144     payload_format: Optional[int] = None,
   145     msg_expiry: Optional[int] = None,
   146     packet_id: int = 1,
   147 ) -> bytes:
   148     """Build an MQTT 5.0 PUBLISH packet with properties."""
   149     first_byte = (PacketType.PUBLISH << 4) | ((qos & 0x03) << 1)
   150     if retain:
   151         first_byte |= 0x01
   152
   153     variable_header = encode_utf8_string(topic)
   154     if qos > 0:
   155         variable_header += encode_uint16(packet_id)
   156
   157     # Properties
   158     props = bytearray()
   159     if payload_format is not None:
   160         props += bytes([0x01, payload_format])
   161     if msg_expiry is not None:
   162         props += bytes([0x02]) + struct.pack("!I", msg_expiry)
   163     if topic_alias is not None:
   164         props += bytes([0x23]) + struct.pack("!H", topic_alias)
   165     if user_props:
   166         for k, v in user_props:
   167             props += bytes([0x26]) + encode_utf8_string(k) +
 encode_utf8_s
       tring(v)
   168
   169     variable_header += encode_remaining_length(len(props)) +
 bytes(props)
   170     remaining = variable_header + payload
   171     return bytes([first_byte]) + encode_remaining_length(len(remaining))
  +
        remaining

   174 def build_subscribe_v5(
   175     topic_filter: str,
   176     packet_id: int = 1,
   177     qos: int = 0,
   178     sub_id: Optional[int] = None,
   179     user_props: Optional[List[Tuple[str, str]]] = None,
   180 ) -> bytes:
   181     """Build an MQTT 5.0 SUBSCRIBE packet."""
   182     # Properties
   183     props = bytearray()
   184     if sub_id is not None:
   185         # Subscription Identifier (0x0B) — variable byte int
   186         props += bytes([0x0B]) + encode_remaining_length(sub_id)
   187     if user_props:
   188         for k, v in user_props:
   189             props += bytes([0x26]) + encode_utf8_string(k) +
 encode_utf8_s
       tring(v)
   190
   191     variable_header = encode_uint16(packet_id)
   192     variable_header += encode_remaining_length(len(props)) +
 bytes(props)
   193
   194     # Topic filter + options byte
   195     options = (qos & 0x03)
   196     payload = encode_utf8_string(topic_filter) + bytes([options])
   197     remaining = variable_header + payload
   198     return bytes([0x82]) + encode_remaining_length(len(remaining)) +
 remai
       ning
